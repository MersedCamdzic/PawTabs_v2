PawTabs v2.2.0 — Chrome Web Store publishing checklist

Repo:       this folder (no git remote yet — set one before tagging)

----------------------------------------------------------------
0. WHAT'S NEW IN 2.2.0 (paste into store "What's new" field)
----------------------------------------------------------------

  • Auto-save snapshots on a schedule (minutes / hours / days).
    Oldest auto-saves are pruned so storage stays lean.
  • Smart restore for large snapshots. Tabs open as instant
    lazy placeholders and only load when clicked — restoring
    90 tabs no longer freezes Chrome.
  • Restore modes: per-window (match original layout), single
    window (everything into one), or reuse-if-exists. Optional
    "Close current windows first" wipes state before restore.
  • Demo mode: try PawTabs on rich sample data without losing
    your current session — PawTabs snapshots your tabs, opens
    the demo set, and restores your originals on exit.
  • Help modal (? in the popup header) walks through every
    icon, tag, note, and session feature.
  • Windows / Tags / Pawed / Pinned views now refresh
    immediately after adding a tag or note.

1. PRE-FLIGHT (do once before any upload)
----------------------------------------------------------------

  [ ] Smoke-test the ZIP locally:
      chrome://extensions → Developer mode → "Load unpacked"
      → select the `dist/` folder (NOT the zip).
      Click around: popup, MC, group actions, snapshots, sessions,
      cleanup wizard, tab details modal, bulk actions.

  [ ] 2.2.0-specific smoke: enable auto-save with a 1-minute
      cadence, confirm the "Last saved" pill updates + oldest
      auto-session is pruned when max is reached, then disable
      and confirm no more auto-sessions appear.

  [ ] 2.2.0-specific smoke: restore a snapshot in each of the
      three modes; confirm tabs come up as lazy placeholders
      and only navigate when clicked. Try "Close current
      windows first" and confirm the wipe happens.

  [ ] 2.2.0-specific smoke: from Mission Control → Demo, click
      "Enter demo mode", confirm your tabs are backed up and
      demo windows open. Click "Exit demo mode" and confirm
      originals restore.

  [ ] 2.2.0-specific smoke: click ? in popup header, verify
      Help modal opens and every section renders.

  [ ] Confirm window-color theming still works in popup group headers
      AND in tab details "Move to window" picker.

  [ ] Open chrome://extensions/shortcuts and verify Alt+Shift+P
      opens the popup (and that no other shortcut is set).

  [ ] (Optional) Take fresh screenshots — see SCREENSHOTS section
      below. Web Store recommends 1280×800 PNGs, max 5.

  [ ] Verify Privacy Policy URL is reachable. If you don't have a
      public URL yet, host marketing/PRIVACY_POLICY.md somewhere
      free first (GitHub Pages / Gist raw / personal site).

CANDIDATE IDEAS FOR 2.2.0+ (not committed, just notes)
================================================================

  Polish:
    - Persist popup group-header collapse state per-grouping
      (currently keyed only by group key — switching grouping
      shows everything open again).
    - Custom in-app toast for keyboard-triggered actions instead
      of the action-icon badge flash (the badge flash was added
      back when keyboard shortcuts existed; if shortcuts return,
      tighten the visual).
    - Cleanup Wizard: show a per-tab "why this rule triggered"
      hint on hover.

  Features:
    - Window-level shortcuts (Mac users — investigate Cmd+Shift+P
      etc. that don't conflict with Chrome native).
    - Tag autocomplete in Tab Details modal (suggest from existing
      taggedUrls).
    - Per-window "default new-tab tags" (any tab opened in a named
      window inherits that window's tag).
    - Export/import all data as JSON (one-click backup outside of
      Wizard backups).
    - Light/dark theme switcher (currently follows system tokens).

  Tech debt:
    - Lift PawedView/TagsView filtering into MissionControl so the
      bulk URL menu doesn't depend on view callbacks.
    - Consolidate BulkActionsMenu + BulkUrlActionsMenu + GroupActions
      into one configurable component (lots of duplication).
    - Replace `confirm()` calls with the existing Modal component
      (less jarring than native alert).
